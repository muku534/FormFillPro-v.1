# Screenshots Guide for Store Submission

This guide helps you create professional screenshots for the Chrome Web Store and Edge Add-ons listings.

## Specifications

### Size Requirements
- **Recommended:** 1280x800 pixels (16:10 aspect ratio)
- **Alternative:** 640x400 pixels
- **Format:** PNG or JPEG
- **Maximum file size:** 5MB per image
- **Number needed:** 5-10 screenshots

---

## Screenshot Ideas

### 1. Main Popup Interface
**Focus:** Extension popup with "Fill All Forms" button

**What to show:**
- Clean popup interface
- "Fill All Forms" button prominently displayed
- Quick Fill buttons (Email, Phone, Name, etc.)
- Detected fields counter

**Page to use:** Any form page (e.g., contact form, signup page)

**Steps:**
1. Open the extension popup
2. Make sure form fields are detected
3. Capture the popup window with the page partially visible in background

---

### 2. Form Detection in Action
**Focus:** Extension showing detected form fields

**What to show:**
- Extension popup open
- "DETECTED FIELDS" section expanded
- List of detected fields with their types
- A real form visible on the page

**Page to use:** A form with multiple field types (name, email, phone, message)

**Steps:**
1. Navigate to a contact or registration form
2. Open extension popup
3. Show the detected fields list
4. Capture both the popup and the form

---

### 3. Before/After Comparison
**Focus:** Empty form vs. filled form

**Option A - Side by Side:**
- Left side: Empty form
- Right side: Same form filled with data
- Use an image editor to create split view

**Option B - Sequential:**
- Screenshot 3a: Empty form
- Screenshot 3b: Same form filled

**Page to use:** Contact form or signup form

**Steps:**
1. Capture empty form
2. Click "Fill All Forms"
3. Capture filled form
4. Combine or submit separately

---

### 4. Quick Fill Buttons in Action
**Focus:** Individual field-type fill buttons

**What to show:**
- Extension popup showing quick fill buttons
- Highlight one being used (or show result)
- Form field being filled by that specific button

**Page to use:** Simple form with clear field labels

**Steps:**
1. Open extension popup
2. Show the Quick Fill section
3. Optionally: Show one field being filled

---

### 5. Contact Form Example
**Focus:** Real-world use case - filling a contact form

**What to show:**
- Complete contact form (Name, Email, Subject, Message)
- Extension popup visible or badge showing it's active
- Form filled with realistic data

**Page to use:** Any contact form (can create a demo page)

**Steps:**
1. Navigate to contact form
2. Open extension
3. Click "Fill All Forms"
4. Capture the completed form

---

### 6. Settings/Templates View
**Focus:** Extension settings or templates tab

**What to show:**
- Settings interface
- Template options (if implemented)
- Customization options

**Steps:**
1. Open extension popup
2. Click on "Templates" or "Settings" tab
3. Capture the interface

---

### 7. Multiple Field Types
**Focus:** Extension handling various field types

**What to show:**
- Form with diverse fields (email, phone, address, dropdown, date, etc.)
- All fields correctly filled
- Extension popup showing successful fill

**Page to use:** Complex registration or checkout form

---

### 8. Keyboard Shortcut Demo
**Focus:** Showing keyboard shortcut functionality

**What to show:**
- Form page
- Visual indicator of keyboard shortcut (Alt+Shift+F)
- Text overlay or caption: "Press Alt+Shift+F to fill instantly"

**Optional:** Use video editing to add keyboard graphic overlay

## Taking Screenshots

### Tools Recommended

**Windows:**
- Snipping Tool (Built-in)
- ShareX (Free, powerful)
- Lightshot (Free)

**Mac:**
- Command+Shift+4 (Built-in)
- CleanShot X (Paid)
- Monosnap (Free)

**Linux:**
- Flameshot (Free)
- Shutter (Free)
- GNOME Screenshot (Built-in)

**Browser Extensions:**
- Awesome Screenshot
- Nimbus Screenshot
- FireShot

### Tips for Professional Screenshots

1. **Clean Browser**
   - Remove other extensions from toolbar
   - Use a clean theme
   - Clear bookmarks bar or hide it

2. **Proper Zoom**
   - Use 100% zoom (Ctrl/Cmd + 0)
   - Or 110% if text is too small

3. **Realistic Data**
   - Use the extension to generate data
   - Make sure data looks natural
   - Avoid "test@test.com" - use realistic emails

4. **Good Lighting/Contrast**
   - High contrast between elements
   - Clear, readable text
   - No dark mode unless that's your theme

5. **Annotations (Optional)**
   - Add arrows pointing to key features
   - Use red boxes to highlight important elements
   - Add short text labels

6. **Consistency**
   - Use same theme/styling across all screenshots
   - Same browser window size
   - Same form styling (or at least complementary)

## Editing Screenshots

### Recommended Size: 1280x800

If your screenshots aren't exactly this size:

1. **Resize:** Use any image editor
   - Maintain aspect ratio
   - Add padding if needed
   - Center the content

2. **Crop:** Remove unnecessary UI elements
   - Focus on the extension and form
   - Remove OS taskbar unless relevant

3. **Annotate:** Add helpful markers
   - Arrows ‚Üí pointing to features
   - Boxes around important areas
   - Text labels explaining features

### Recommended Tools

- **Canva** (Free, web-based) - Easy annotations
- **Figma** (Free) - Professional design tool
- **Photoshop** (Paid) - Professional editing
- **GIMP** (Free) - Open-source alternative to Photoshop
- **Paint.NET** (Free, Windows) - Simple but powerful

## Screenshot Checklist

Before submitting, verify each screenshot:

- [ ] Is 1280x800 pixels (or 640x400)
- [ ] Is PNG or JPEG format
- [ ] Under 5MB file size
- [ ] Shows actual extension functionality
- [ ] Has clear, readable text
- [ ] Demonstrates a specific feature
- [ ] Uses realistic test data (not "test@test.com")
- [ ] Has clean UI (no extra extensions visible)
- [ ] Is professional and polished
- [ ] Accurately represents the extension

---

## Example Screenshot Order

Submit in this order for best storytelling:

1. **Hero Shot** - Main popup interface (first impression)
2. **Detection** - Form fields being detected
3. **Before** - Empty form
4. **After** - Filled form
5. **Quick Fill** - Individual fill buttons
6. **Contact Form** - Real-world example
7. **Settings** - Customization options (if available)
